    Issue #3184175 by Kingdutch: move from bower to npm
    
    This replaces the packages that were pulled from bower to be now pulled
    from NPM. Bower has been deprecated for a while and is quite unreliable
    when used through asset-packagist. Packages from NPM should do better.
    
    This was tested by committing the installed files to git, changing the
    packages and then looking at the diffs. This revealed that the files
    used in the library definitions are still shipped. The only detected
    change is the renaming of `waves` to `node-waves` because the package
    name is different on NPM.
    
    An observed benefit of this change is that the source and test files are
    no longer included (but the built and minified files are still present
    in the NPM packages).

    Issue #3171512 by agami4, Kingdutch: Add title to SVG elements for all type of the teasers (#2023)
    
    * Issue #3171512 by agami4: Add title to SVG elements for the all type of the teasers
    
    We ensure that all information in a teaser that is conveyed visually is
    also available without being able to see the visual components.
    
    * Move event listener to body to prevent 'clickable' in voice over
    
    When a click event handler exists on a parent of an element that is not
    the body, OS X Voice Over (and possibly other screenreaders) will
    announce any element selected as "clickable". This is quite disruptive
    when going across teasers that contain non-clickable text.
    
    Delegating the execution of the click handler to the body solves this
    issue and no longer makes a string such as "Open Group" announce as
    "Open Group, clickable".
    
    In this case this is the right thing to do since the menu should only be
    closable by clicking on the content on mobile, when the menu is open.
    For screenreaders there are other ways to properly close the menu after
    opening it.
    
    Background: http://www.pauljadam.com/demos/voiceover-clickable.html
    
    * Apply screenreader fixes to non landing-page and sky teasers

    simple `if` check to  be false. This is not a problem in the normal
    layout because the content is limited to 67% width and this region just
    fills the remaining 1/3rd of the page.

    left which causes the other content to be shifted to the right even if
    there appears to be no actual content. To solve this we render the
    contents of the region in the if statement. This is slightly slower but
    is the only real way to find out if anything other than whitespace will
    be rendered.
