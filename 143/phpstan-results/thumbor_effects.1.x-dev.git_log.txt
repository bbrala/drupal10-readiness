    Fix thumbor filters by url-encoding only the source URL path
    
    Instead of rawurlencode'ing all parts of the thumbor URL,
    only the source URL needs to be encoded since it may contain
    characters like "?" and following GET parameters,
    which break signature verification as we saw in #3059977.
    
    Encoding the other parts breaks filters because the ":" in
    "filters:..." do get encoded as "%3A" and thumbor does not detect
    them as command anymore.
    
    This bug was introduced by myself when solving #3059977.
    
    Resolves: https://www.drupal.org/project/thumbor_effects/issues/3284888
