    Simplest approach for implementing cache-tags
    
    * Introduce two backend-wide sets: 'meta/deleted' and 'meta/stale'
    * Upon cache-get also check meta/deleted, meta/stale
    * Add one set per cache-entry containing the tags for the cid
      'meta/tagsByKey:[cid]'
    * Add one set per tag containing the cids associated with it
      'meta/keysByTag:[tag]'
    * When invalidating/deleting per-cid, add the cid to 'meta/stale' or
      'meta/deleted'
    * When invalidating/deleting tags, build a union of 'meta/stale' or
      'meta/deleted' with 'meta/keysByTag:[tag]'
    
    Garbage collection needs to be implemented, basically we can walk
    through 'meta/deleted' and remove stuff.
    
    Also the set-method is not atomic. We might either use watch/multi/exec
    or implement it as a server-side redis-script.
